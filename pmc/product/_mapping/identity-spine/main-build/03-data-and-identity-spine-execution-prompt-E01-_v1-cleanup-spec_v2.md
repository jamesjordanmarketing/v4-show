## Governing Policies

These three policies apply to every decision in this spec. When in doubt, they take precedence.

**Policy 1 — Keep when uncertain.**
If we are unsure whether a table, column, or file is in use, we keep it. Removing something that turns out to be needed is far more costly than keeping something unused. Cleanup can always be done in a future iteration once functionality is confirmed intact.

**Policy 2 — System config tables are never cleared.**
"Meta" tables that contain the definitions, configurations, templates, or reference data the system needs to operate are preserved even through a full data reset. These include: `templates`, `prompt_templates`, `personas`, `emotional_arcs`, `training_topics`, `evaluation_prompts`, `pipeline_base_models`, `pipeline_base_models`.

**Policy 3 — Audit before clearing any prompt/config data.**
Prompts and configuration rows fall into two categories: (a) reusable templates used across many operations → keep, (b) run-specific records generated for a single execution → clear. Do not clear any prompt table without first auditing which category each row belongs to.

> **SAOL requirement:** ALL database operations use SAOL (the Supabase Agent Ops Library). No raw `supabase-js`, no `psql` directly.

## Section A — Drop `kv_store_*` Tables

### What Are They?

35 tables named `kv_store_<hash>` were auto-generated by **Figma Make** — a Supabase plugin used during early UI prototyping. Each session of Figma Make creates its own uniquely-named key-value table:

```sql
CREATE TABLE kv_store_<hash> (
  key   TEXT  NOT NULL PRIMARY KEY,
  value JSONB NOT NULL
);
```

They are isolated UI state stores and have no connection to the Next.js application.

### Codebase References?

**Zero.** The only file that mentions `kv_store` is `src/supabase/functions/server/kv_store.tsx` — an autogenerated Supabase Edge Function file, not part of the Next.js app. It references a table called `kv_store_6b04c628` which is not even one of the 35 tables in the database (that session's table was already dropped). No file under `src/app/`, `src/lib/`, `src/components/`, or `src/hooks/` references `kv_store`.

### Safe to Drop?

Yes. Zero risk. Combined size ~2.2 MB.

### Summary

- `conversations` → empty
- `batch_jobs` → empty
- `documents` → empty (also dropped entirely in Section C)

The `_orphaned_records` table is truncated as part of Section E. E04's NOT NULL constraint on `user_id` will then apply to empty tables and succeed cleanly.

**No orphan user account is needed. All affected data is test-era trash that Section E removes.**

// scripts/cleanup/E2-global-reset-selective.js

require('dotenv').config({ path: '../.env.local' });
const saol = require('.');

const PRESERVE_TF_ID  = 'e57e3266-3acc-4252-8b92-b165993076fb'; // training_files: "Batch 6- 12 conversations #1"
const PRESERVE_DS_ID  = '8ec1aba1-6618-45ad-b10c-1970ee95b9ec'; // datasets: "Batch 6- 12 conversations #1"
const PRESERVE_JOB_ID = '6fd5ac79-c54b-4927-8138-ca159108bcae'; // pipeline_training_jobs: "Batch-6-thru-12_v1"

(async () => {
  // --- Preview preserved records ---
  console.log('\n=== VERIFYING PRESERVED RECORDS EXIST ===');
  const check = await saol.agentExecuteSQL({ sql: `
    SELECT 'training_file' AS type, id, name FROM training_files WHERE id = '${PRESERVE_TF_ID}'
    UNION ALL
    SELECT 'dataset', id, name FROM datasets WHERE id = '${PRESERVE_DS_ID}'
    UNION ALL
    SELECT 'pipeline_job', id, job_name FROM pipeline_training_jobs WHERE id = '${PRESERVE_JOB_ID}'
  `, transport: 'pg' });
  console.log('Records to preserve:');
  check.rows.forEach(r => console.log(` ${r.type}: ${r.name || r.job_name} (${r.id})`));

  const linkedConvCount = await saol.agentExecuteSQL({ sql: `
    SELECT COUNT(*) AS cnt FROM training_file_conversations WHERE training_file_id = '${PRESERVE_TF_ID}'
  `, transport: 'pg' });
  console.log(` conversations to preserve: ${linkedConvCount.rows[0].cnt}`);

  // --- Verify user_id is set on preserved records (Identity Spine check) ---
  const idCheck = await saol.agentExecuteSQL({ sql: `
    SELECT 'training_file' AS type, COUNT(*) FILTER (WHERE user_id IS NULL) AS null_user_id
    FROM training_files WHERE id = '${PRESERVE_TF_ID}'
    UNION ALL
    SELECT 'dataset', COUNT(*) FILTER (WHERE user_id IS NULL)
    FROM datasets WHERE id = '${PRESERVE_DS_ID}'
    UNION ALL
    SELECT 'pipeline_job', COUNT(*) FILTER (WHERE user_id IS NULL)
    FROM pipeline_training_jobs WHERE id = '${PRESERVE_JOB_ID}'
    UNION ALL
    SELECT 'linked_conversations', COUNT(*) FILTER (WHERE c.user_id IS NULL)
    FROM conversations c
    INNER JOIN training_file_conversations tfc ON tfc.conversation_id = c.id
    WHERE tfc.training_file_id = '${PRESERVE_TF_ID}'
  `, transport: 'pg' });

  idCheck.rows.forEach(r => console.log(` ${r.type}: null_user_id = ${r.null_user_id}`));

  const hasNullUserId = idCheck.rows.some(r => parseInt(r.null_user_id) > 0);
  if (hasNullUserId) {
    console.error('\n⚠ WARNING: Some preserved records have null user_id. E04 NOT NULL constraint will fail for these rows.');
    console.error('Resolve by setting user_id on the affected rows before running E04.');
  } else {
    console.log('\n✓ All preserved records have valid user_id — Identity Spine safe.');

    DELETE FROM pipeline_evaluation_runs WHERE job_id != '${PRESERVE_JOB_ID}';
    DELETE FROM pipeline_training_metrics WHERE job_id != '${PRESERVE_JOB_ID}';
    DELETE FROM model_artifacts;
    DELETE FROM training_jobs;
    DELETE FROM pipeline_training_jobs WHERE id != '${PRESERVE_JOB_ID}';
    DELETE FROM pipeline_inference_endpoints WHERE job_id != '${PRESERVE_JOB_ID}';

    -- Training + Datasets: keep the preserved pair
    DELETE FROM training_file_conversations WHERE training_file_id != '${PRESERVE_TF_ID}';
    DELETE FROM datasets WHERE id != '${PRESERVE_DS_ID}';
    DELETE FROM training_files WHERE id != '${PRESERVE_TF_ID}';

    -- Conversations: keep the 242 linked to the preserved training file
    DELETE FROM batch_checkpoints;
    DELETE FROM batch_items;
    DELETE FROM batch_jobs;
    DELETE FROM generation_logs;
    DELETE FROM failed_generations;
    DELETE FROM conversation_turns
      WHERE conversation_id NOT IN (
        SELECT conversation_id FROM training_file_conversations WHERE training_file_id = '${PRESERVE_TF_ID}'
      );
    DELETE FROM legacy_conversation_turns;
    DELETE FROM multi_turn_conversations;
    DELETE FROM conversations
      WHERE id NOT IN (
        SELECT conversation_id FROM training_file_conversations WHERE training_file_id = '${PRESERVE_TF_ID}'
